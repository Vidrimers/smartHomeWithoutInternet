## 1. Что такое ESP32-S3-BOX-3B

ESP32-S3-BOX-3B — это готовое устройство от Espressif с:
- 🎤 Встроенным микрофоном
- 🔊 Встроенным динамиком
- 📺 Сенсорным экраном 2.4"
- 📡 Wi-Fi + Bluetooth

**Отличие BOX-3B от BOX-3:** BOX-3B — упрощённая версия без дополнительных аксессуаров (датчик температуры, ИК-передатчик). Для голосового ассистента BOX-3B полностью достаточен.

**Как работает:**
```
Ты говоришь "OK Nabu" → ESP32 слышит wake word
→ Записывает команду → Отправляет в Whisper (на сервере)
→ Whisper распознаёт текст → Assist обрабатывает команду
→ HA выполняет действие → Piper озвучивает ответ
→ ESP32 воспроизводит ответ через динамик
```

## 2. Что нужно перед началом

- ✅ Home Assistant запущен (у тебя уже есть на `http://10.0.0.3:8123`)
- ✅ ESP32-S3-BOX-3B подключён к серверу по USB-C
- ✅ **Chrome или Edge браузер** на компьютере (не телефон, не Firefox!)
- ✅ Знаешь пароль от Wi-Fi сети 2.4 GHz

> ⚠️ Прошивка устанавливается только через **Chromium-браузер** (Chrome или Edge) на **компьютере**. Firefox и мобильные браузеры не поддерживают Web Serial API.

## 3. Шаг 1: Запустить Whisper и Piper (Docker)

У тебя HA в Docker, поэтому Whisper и Piper запускаются как отдельные контейнеры.

### Запусти Whisper (распознавание речи):

```bash
docker run -d \
  --name whisper \
  --restart=unless-stopped \
  -p 10300:10300 \
  -v ~/whisper:/data \
  rhasspy/wyoming-whisper \
  --model small-int8 \
  --language ru
```

> **Модели Whisper:**
> - `tiny-int8` — быстро, качество слабое
> - `small-int8` — хорошее качество, рекомендуется для N100
> - `medium-int8` — отличное качество, чуть медленнее

### Запусти Piper (синтез речи):

```bash
docker run -d \
  --name piper \
  --restart=unless-stopped \
  -p 10200:10200 \
  -v ~/piper:/data \
  rhasspy/wyoming-piper \
  --voice ru_RU-ruslan-medium
```

### Запусти OpenWakeWord (распознавание wake word на сервере):

```bash
docker run -d \
  --name openwakeword \
  --restart=unless-stopped \
  -p 10400:10400 \
  rhasspy/wyoming-openwakeword \
  --preload-model ok_nabu
```

### Проверь что контейнеры запустились:

```bash
docker ps | grep -E "whisper|piper|openwakeword"
```

Должны быть все три в статусе `Up`.

## 4. Шаг 2: Настроить голосовой пайплайн в HA

### Добавь Wyoming интеграции в HA:

1. Settings → Devices & Services → **Add Integration**
2. Найди **Wyoming Protocol**
3. Добавь три раза (для каждого сервиса):

| Сервис | Host | Port |
|--------|------|------|
| Whisper | `127.0.0.1` | `10300` |
| Piper | `127.0.0.1` | `10200` |
| OpenWakeWord | `127.0.0.1` | `10400` |

### Создай голосовой пайплайн:

1. Settings → **Voice Assistants**
2. Нажми **Add Assistant**
3. Заполни:
   - **Name:** `Локальный ассистент`
   - **Language:** `Russian`
   - **Speech-to-text:** `faster-whisper` (Whisper)
   - **Text-to-speech:** `piper` (Piper)
   - **Wake word:** `ok_nabu` (OpenWakeWord)
4. Нажми **Create**

## 5. Шаг 3: Прошить ESP32-S3-BOX-3B

> ⚠️ Используй **Chrome или Edge** на компьютере. Подключи ESP32-S3-BOX-3B к компьютеру по USB-C.

### Способ 1: Через официальный сайт HA (рекомендуется)

1. Открой в Chrome/Edge: **[https://www.home-assistant.io/voice_control/s3_box_voice_assistant/](https://www.home-assistant.io/voice_control/s3_box_voice_assistant/)**
2. Найди раздел **"Using the ESP32-S3-BOX-3(B)"**
3. Нажми кнопку **Connect**
4. В появившемся окне **пока не подключай** ESP32 — сначала посмотри список портов
5. Подключи USB-C кабель к ESP32-S3-BOX-3B (не к докстанции!)
6. В списке появится новый порт — выбери его и нажми **Connect**
7. Нажми **Install Voice Assistant** → **Install**
8. Дождись окончания установки (~2-3 минуты)

**Если ESP32 не появляется в списке портов:**

Нужно войти в режим прошивки вручную:
1. Зажми кнопку **Boot** (левая верхняя кнопка)
2. Не отпуская Boot, нажми кнопку **Reset** (левая нижняя кнопка)
3. Отпусти обе кнопки
4. Теперь ESP32 должен появиться в списке

## 6. Шаг 4: Подключить к Wi-Fi и добавить в HA

После прошивки:

1. Нажми **Next**
2. Выбери свою Wi-Fi сеть **2.4 GHz** из списка
3. Введи пароль
4. Нажми **Connect**
5. После подключения нажми **Add to Home Assistant**

Откроется ссылка на HA. Если HA не открывается автоматически:

1. Открой `http://10.0.0.3:8123`
2. Settings → Devices & Services
3. Должно появиться уведомление о новом устройстве **ESP32-S3-BOX-3**
4. Нажми **Configure** → **Submit**
5. Добавь в комнату → **Finish**

## 7. Шаг 5: Настроить wake word

После добавления в HA:

1. Settings → Devices & Services → **ESPHome**
2. Нажми на **ESP32-S3-BOX-3** → **Device**
3. Найди раздел **Configuration**:

### Выбери пайплайн:

- **Assist pipeline:** выбери `Локальный ассистент` (созданный в Шаге 2)

### Выбери где обрабатывать wake word:

| Вариант | Описание | Рекомендация |
|---------|----------|--------------|
| **On device** | Wake word обрабатывается на ESP32 | ✅ Быстрее, приватнее |
| **On server** | Wake word обрабатывается на сервере (OpenWakeWord) | Медленнее |

Рекомендуется **On device** — ESP32-S3 достаточно мощный для локальной обработки.

### Доступные wake words (On device):

- **OK Nabu** (по умолчанию)
- **Hey Jarvis**
- **Alexa**

## 8. Шаг 6: Проверить работу

1. Скажи **"OK Nabu"**
2. Экран ESP32 должен стать **синим** — устройство слушает
3. Скажи команду, например: **"Включи свет в кухне"**
4. Экран станет **зелёным** — команда выполнена
5. ESP32 озвучит ответ через динамик

### Примеры команд:

```
"OK Nabu, включи свет в кухне"
"OK Nabu, выключи все лампочки"
"OK Nabu, какая температура в комнате?"
"OK Nabu, который час?"
"OK Nabu, включи сцену Кино"
```

> **Важно:** Используй названия комнат и устройств **точно так же**, как они называются в HA.

## 11. Troubleshooting

### ESP32 не появляется в списке портов браузера

- Убедись что используешь **Chrome или Edge** (не Firefox)
- Попробуй другой USB-C кабель (некоторые кабели только для зарядки)
- Войди в режим прошивки вручную: зажми **Boot** + нажми **Reset**
- Попробуй другой USB порт на компьютере

---

### Устройство добавлено в HA, но не реагирует на wake word

1. Проверь что пайплайн выбран правильно (Settings → ESPHome → Device → Assist pipeline)
2. Проверь что Whisper запущен: `docker ps | grep whisper`
3. Проверь логи Whisper: `docker logs whisper`
4. Убедись что **Mute** выключен в настройках устройства

---

### Whisper не распознаёт русский язык

Проверь что контейнер запущен с флагом `--language ru`:

```bash
docker inspect whisper | grep -A 5 "Cmd"
```

Если нет — пересоздай контейнер:

```bash
docker stop whisper
docker rm whisper
docker run -d \
  --name whisper \
  --restart=unless-stopped \
  -p 10300:10300 \
  -v ~/whisper:/data \
  rhasspy/wyoming-whisper \
  --model small-int8 \
  --language ru
```

---

### Piper говорит не по-русски или не говорит вообще

Проверь логи:

```bash
docker logs piper
```

Пересоздай с правильным голосом:

```bash
docker stop piper
docker rm piper
docker run -d \
  --name piper \
  --restart=unless-stopped \
  -p 10200:10200 \
  -v ~/piper:/data \
  rhasspy/wyoming-piper \
  --voice ru_RU-ruslan-medium
```

---

### Команды не выполняются (HA не понимает)

- Проверь что устройства добавлены в **Areas** (комнаты) в HA
- Используй точные названия комнат из HA
- Проверь Developer Tools → Assist — там можно тестировать команды текстом

## 12. Кастомные фразы и падежи

По умолчанию HA Assist ищет **точное название** устройства. "Люстра" — найдёт. "Люстру" или "люстру в комнате" — не найдёт.

Есть два способа научить ассистента понимать разные формулировки.

---

### Способ 1: Sentence trigger в автоматизации (проще)

Это официальный рекомендуемый способ из документации HA.

1. Settings → Automations & Scenes → **Create Automation**
2. В поле **Trigger** выбери **Sentence**
3. Добавь несколько вариантов фраз:

```
включи люстру
включи люстру в комнате
зажги люстру
свет в комнате включи
```

4. В **Action** добавь действие: `light.turn_on` → выбери нужную лампу
5. Опционально: добавь **Set conversation response** — ответ ассистента:
   ```
   Люстра включена!
   ```
6. Сохрани

**Пример для выключения:**

Создай отдельную автоматизацию с фразами:
```
выключи люстру
выключи люстру в комнате
погаси люстру
свет в комнате выключи
```

> **Важно:** Не используй знаки препинания в фразах. Можно добавить сколько угодно вариантов — все они будут триггерить одну автоматизацию.

---

### Способ 2: Кастомные предложения через YAML файл (мощнее)

Позволяет создавать шаблонные фразы с переменными. Подходит для управления несколькими устройствами одним шаблоном.

**Шаг 1:** Создай папку и файл:

```bash
mkdir -p ~/homeassistant/custom_sentences/ru
nano ~/homeassistant/custom_sentences/ru/lights.yaml
```

**Шаг 2:** Вставь содержимое:

```yaml
language: "ru"
intents:
  HassTurnOn:
    data:
      - sentences:
          - "включи [люстру|свет|лампу] [в комнате|в кухне|в коридоре|в спальне]"
          - "зажги [люстру|свет|лампу] [в (комнате|кухне|коридоре)]"
  HassTurnOff:
    data:
      - sentences:
          - "выключи [люстру|свет|лампу] [в комнате|в кухне|в коридоре|в спальне]"
          - "погаси [люстру|свет|лампу] [в (комнате|кухне|коридоре)]"
```

Сохрани: `Ctrl+O`, `Enter`, `Ctrl+X`

**Шаг 3:** Перезапусти HA:

```bash
docker restart homeassistant
```

> **Примечание:** Этот способ расширяет встроенные интенты `HassTurnOn` и `HassTurnOff`. HA всё равно будет искать устройство по имени в указанной комнате — убедись что устройства правильно добавлены в Areas.

---

### Способ 3: Псевдонимы устройств (самый простой)

В HA можно добавить **псевдонимы** к устройствам — дополнительные имена по которым Assist их найдёт.

1. Settings → Devices & Services → найди устройство
2. Нажми на устройство → **Edit**
3. В поле **Aliases** добавь варианты:
   - `люстра`
   - `свет в комнате`
   - `лампа`
4. Сохрани

Теперь можно говорить любой из псевдонимов и HA найдёт устройство.

---

### Сравнение способов:

| Способ | Сложность | Гибкость | Когда использовать |
|--------|-----------|----------|-------------------|
| **Sentence trigger** | Простой | Средняя | Конкретные фразы для одного устройства |
| **YAML файл** | Средний | Высокая | Шаблоны для многих устройств |
| **Псевдонимы** | Очень простой | Низкая | Добавить пару альтернативных имён |

## 13. Внешние колонки для голосовых ответов

Встроенный динамик ESP32-S3-BOX-3B небольшой. Если хочешь лучший звук — можно вывести голосовые ответы Piper на внешние колонки.

---

### Вариант 1: Колонки через 3.5mm разъём мини-ПК (рекомендуется)

Самый простой и надёжный способ. Piper синтезирует речь на сервере и воспроизводит через звуковую карту мини-ПК.

#### Шаг 1: Подключи колонки к мини-ПК

Воткни колонки в 3.5mm разъём мини-ПК.

#### Шаг 2: Проверь звуковые устройства на сервере

```bash
# Установи утилиты ALSA
sudo apt install alsa-utils

# Посмотри список устройств вывода
aplay -l

# Тест звука (Ctrl+C чтобы остановить)
speaker-test -t wav -c 2
```

Запомни номер карты и устройства из вывода `aplay -l`, например:
```
card 0: PCH [HDA Intel PCH], device 0: ALC... [...]
```

#### Шаг 3: Установи VLC на сервере

```bash
sudo apt install vlc
```

#### Шаг 4: Добавь VLC как медиаплеер в HA

Установи интеграцию **VLC media player** через HACS или вручную добавь в `configuration.yaml`:

```bash
nano ~/homeassistant/configuration.yaml
```

Добавь:

```yaml
media_player:
  - platform: vlc_telnet
    name: "Колонки сервера"
    host: 127.0.0.1
    port: 4212
    password: vlcpassword
```

Запусти VLC в режиме telnet на сервере:

```bash
# Запусти VLC как сервис
cvlc --intf telnet --telnet-password vlcpassword --telnet-port 4212 &
```

Или через Docker:

```bash
docker run -d \
  --name vlc \
  --restart=unless-stopped \
  --network=host \
  --device /dev/snd \
  -e PUID=1000 \
  -e PGID=1000 \
  linuxserver/vlc
```

#### Шаг 5: Настрой пайплайн на воспроизведение через VLC

В голосовом пайплайне (Settings → Voice Assistants → твой пайплайн):
- **Text-to-speech:** piper
- **TTS output device:** выбери `media_player.колонки_сервера`

#### Шаг 6: Проверь через Developer Tools

Settings → Developer Tools → Actions → `tts.speak`:
```yaml
entity_id: media_player.колонки_сервера
message: "Привет, это тест"
```

---

### Вариант 2: Bluetooth колонка

> ⚠️ Bluetooth на Ubuntu Server сложнее в настройке, чем 3.5mm. Рекомендуется только если нет проводного варианта.

#### Шаг 1: Установи Bluetooth утилиты

```bash
sudo apt install bluez pulseaudio pulseaudio-module-bluetooth
```

#### Шаг 2: Подключи колонку

```bash
# Запусти bluetoothctl
bluetoothctl

# Включи Bluetooth
power on
agent on

# Сканируй устройства
scan on

# Когда увидишь свою колонку (запомни MAC адрес)
scan off

# Подключись (замени MAC на свой)
pair XX:XX:XX:XX:XX:XX
connect XX:XX:XX:XX:XX:XX
trust XX:XX:XX:XX:XX:XX

exit
```

#### Шаг 3: Проверь подключение

```bash
# Проверь что колонка видна как аудио устройство
pactl list sinks short
```

#### Шаг 4: Настрой автоподключение

```bash
# Добавь в автозапуск
sudo systemctl enable bluetooth
```

#### Шаг 5: Используй как медиаплеер в HA

Аналогично Варианту 1 — через VLC или другой медиаплеер, но указав Bluetooth устройство как выход.

---

### Сравнение вариантов:

| Вариант | Сложность | Надёжность | Качество звука |
|---------|-----------|------------|----------------|
| **3.5mm (проводной)** | Простая | ✅ Высокая | Зависит от колонок |
| **Bluetooth** | Сложная | ⚠️ Средняя (могут быть разрывы) | Зависит от колонок |
| **Встроенный динамик ESP32** | Нет настройки | ✅ Высокая | ❌ Слабый |

**Рекомендация:** используй проводные колонки через 3.5mm — это проще и надёжнее.
