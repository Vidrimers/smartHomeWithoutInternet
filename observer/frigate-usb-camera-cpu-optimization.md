# 🔧 Оптимизация USB камер в Frigate — снижение нагрузки CPU

Краткое руководство по снижению нагрузки CPU для USB веб-камер в Frigate.

---

## 📊 Проблема

USB камеры используют **15-25% CPU** (в FFmpeg), что значительно больше чем IP камеры (3-5%).

**Причины:**
- Формат MJPEG (каждый кадр = отдельное JPEG изображение)
- Нет аппаратного ускорения (`hwaccel_args: []`)
- Один поток для детекции И записи

---

## ✅ Решение

### 1. Проверь текущую конфигурацию

```bash
nano ~/frigate/config/config.yml
```

Найди секцию с USB камерой (например, `home_usb` или `indoor_usb`).

---

### 2. Оптимизированная конфигурация (рекомендуется)

```yaml
home_usb:
  ffmpeg:
    hwaccel_args: []
    inputs:
      - path: /dev/video0
        input_args: -f v4l2 -input_format mjpeg -video_size 640x480 -framerate 20
        roles:
          - detect
          - record
  detect:
    width: 640
    height: 480
    fps: 10  # Анализировать 10 кадров из 20
    enabled: true
  objects:
    track:
      - person
  record:
    enabled: true
    continuous:
      days: 0
    motion:
      days: 3
  snapshots:
    enabled: true
    retain:
      default: 7
```

**Изменения:**
- Разрешение: `640x480` (вместо 1280x720)
- Framerate: `20 fps` (вместо 30)
- FPS детекции: `10` (вместо 15)

**Результат:** CPU снизится с **21%** до **~8-10%**

---

### 3. Применить изменения

```bash
cd ~/frigate
docker compose restart
```

Проверь нагрузку через 1-2 минуты в Frigate UI:
```
System → Stats → Camera Processes
```

---

## ⚠️ Важно: Разрешение должно совпадать

**Нельзя делать так:**

```yaml
# ❌ НЕПРАВИЛЬНО
input_args: ... -video_size 1280x720 ...
detect:
  width: 640   # Не совпадает!
  height: 480
```

**Правильно:**

```yaml
# ✅ ПРАВИЛЬНО
input_args: ... -video_size 640x480 ...
detect:
  width: 640   # Совпадает!
  height: 480
```

**Почему:** Frigate получает кадры с разрешением из `input_args` без изменения размера.

---

## 📊 Сравнение вариантов

| Параметр | 640x480 @ 20fps | 1280x720 @ 20fps | 1920x1080 @ 20fps |
|----------|-----------------|------------------|-------------------|
| CPU FFmpeg | ~8-10% | ~15-18% | ~25-30% |
| Качество записи | Среднее | Хорошее | Отличное |
| Качество детекции | ✅ Отлично | ✅ Отлично | ✅ Отлично |
| Место на диске (сутки) | ~300-500 MB | ~1-1.5 GB | ~2-3 GB |

**Вывод:** Для детекции человека **640x480 @ 20fps достаточно**.

## 🔍 Как проверить поддерживаемые форматы камеры

```bash
# Установи утилиты
sudo apt install v4l-utils

# Проверь форматы
v4l2-ctl --list-formats-ext -d /dev/video0
```

Ищи формат **MJPEG** — он лучше всего подходит для Frigate.

## 💡 Дополнительные способы снижения CPU

### 1. Снизить FPS детекции

```yaml
detect:
  fps: 5  # Вместо 10 (для статичных сцен)
```

**Экономия:** ~20% CPU

---

### 2. Отключить запись (если не нужна)

```yaml
record:
  enabled: false
```

**Экономия:** ~30% CPU

---

### 3. Использовать Google Coral TPU

Google Coral USB Accelerator ускоряет **детекцию объектов** (не FFmpeg).

**Без Coral:**
- FFmpeg: 21% CPU
- Детекция: 15% CPU
- **Итого: 36% CPU**

**С Coral:**
- FFmpeg: 21% CPU
- Детекция: 2% CPU
- **Итого: 23% CPU**

## ❓ FAQ

### Почему USB камера использует больше CPU чем IP камера?

- **IP камера:** H.264 (межкадровое сжатие) → 3-5% CPU
- **USB камера:** MJPEG (каждый кадр отдельно) → 15-25% CPU

### Можно ли использовать H.264 для USB камеры?

Некоторые USB камеры поддерживают H.264, но большинство используют только MJPEG.

Проверь:
```bash
v4l2-ctl --list-formats-ext -d /dev/video0 | grep H264
```

Если есть H.264, используй:
```yaml
input_args: -f v4l2 -input_format h264 -video_size 1280x720 -framerate 30
```

### Достаточно ли 640x480 для детекции?

**Да!** Для детекции человека 640x480 более чем достаточно. Высокое разрешение нужно только для рассмотрения мелких деталей (номера машин, лица).

### Почему 20 fps вместо 30?

- Для детекции человека 10 fps достаточно (человек не телепортируется)
- Framerate 20 → анализ 10 fps = экономия 33% CPU
- Запись в 20 fps всё ещё плавная (кино снимают в 24 fps)
