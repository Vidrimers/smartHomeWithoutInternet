## Что такое retain?

`retain` — это настройка **сколько дней хранить** данные перед автоматическим удалением.

Frigate хранит два типа данных:
- 📹 **Видеозаписи** (`.mp4`) — управляются через секцию `record`
- 📷 **Снимки** (`.jpg`) — управляются через секцию `snapshots`

---

## Структура хранения

```
Frigate данные
├── 📹 Видеозаписи (record)
│   ├── continuous   → постоянная запись
│   ├── motion       → запись при любом движении
│   ├── detections   → запись при обнаружении объекта (person/car/dog)
│   └── alerts       → запись при объекте в заданной зоне
│
└── 📷 Снимки (snapshots)
    └── retain
        └── default  → сколько дней хранить фото
```

## Секция record (видеозаписи)

### Режимы записи:

| Режим | Что триггерит запись | Место на диске |
|-------|---------------------|----------------|
| `continuous` | Всегда, 24/7 | 🔴 Огромное |
| `motion` | Любое движение пикселей (листья, тени, дождь) | 🟡 Большое |
| `detections` | Frigate распознал объект (person, car, dog, cat) | 🟢 Среднее |
| `alerts` | Объект обнаружен в заданной зоне | 🟢 Минимальное |

### Пример конфига:

```yaml
record:
  enabled: true          # Включить запись
  continuous:
    days: 0              # 0 = не писать постоянно
  motion:
    days: 0              # 0 = не писать при любом движении
  detections:
    retain:
      days: 3            # Хранить видео с объектами 3 дня
  alerts:
    retain:
      days: 7            # Хранить видео с объектами в зоне 7 дней
```

### Что происходит с видео:

```
День 1: Frigate обнаружил человека → записал видео
День 2: видео хранится
День 3: видео хранится
День 4: видео автоматически удаляется ✓
```

## Секция snapshots (снимки)

Снимки — это **фотографии** обнаруженных объектов (не видео).

```yaml
snapshots:
  enabled: true          # Включить снимки
  retain:
    default: 7           # Хранить снимки всех объектов 7 дней
    objects:             # Можно задать разное время для разных объектов
      person: 14         # Людей хранить 14 дней
      car: 7             # Машины 7 дней
      dog: 3             # Собак 3 дня
```

### Что происходит со снимками:

```
День 1: Frigate сфотографировал человека → сохранил .jpg
...
День 14: фото автоматически удаляется ✓
```

## Полный пример конфига для экономии места

```yaml
cameras:
  camera_outside_1:
    ffmpeg:
      inputs:
        - path: rtsp://user:pass@IP:554/Streaming/Channels/102
          roles:
            - detect
        - path: rtsp://user:pass@IP:554/Streaming/Channels/101
          roles:
            - record
    detect:
      width: 640
      height: 360
      fps: 10
      enabled: true
    objects:
      track:
        - person
        - car
        - dog
        - cat
    record:
      enabled: true
      continuous:
        days: 0          # Не пишем постоянно
      motion:
        days: 0          # Не пишем при любом движении (листья, тени)
      detections:
        retain:
          days: 3        # Видео с объектами — 3 дня
      alerts:
        retain:
          days: 7        # Видео с объектами в зоне — 7 дней
    snapshots:
      enabled: true
      retain:
        default: 7       # Снимки — 7 дней
```

## Сравнение режимов по месту на диске

Пример: 3 камеры, активная улица, сутки записи

| Конфигурация | Место в сутки | Место за 7 дней |
|--------------|--------------|-----------------|
| `continuous` | ~50 GB | ~350 GB |
| `motion` | ~15-30 GB | ~100-200 GB |
| `detections` | ~1-3 GB | ~7-20 GB |
| `alerts` (только зона) | ~0.1-0.5 GB | ~1-3 GB |

> **Вывод:** Использование `detections` вместо `motion` экономит в 10-20 раз больше места!

## Частые вопросы

### Почему быстро заканчивается место?

Скорее всего включён режим `motion`. Он реагирует на:
- 🍃 Листья на ветру
- ☁️ Облака и тени
- 💡 Изменение освещения
- 🌧️ Дождь
- 🐛 Насекомых рядом с камерой

**Решение:** замени `motion` на `detections`.

---

### Как удалить все записи вручную?

```bash
# Останови Frigate
cd ~/frigate
docker compose stop

# Удали записи
rm -rf ~/frigate/media/recordings/*
rm -rf ~/frigate/media/clips/*

# Запусти снова
docker compose up -d
```

---

### Как проверить сколько места занято?

```bash
du -sh ~/frigate/media/
du -sh ~/frigate/media/recordings/
du -sh ~/frigate/media/clips/
```

### Где физически хранятся файлы?

```
~/frigate/media/
├── recordings/          # Видеозаписи (.mp4)

│   │       └── 19/      # Час (UTC)

│   ├── camera_outside_2/
│   └── home_usb/

```

## Секция motion (настройка детектора движения)

`motion` — это не режим записи, а **настройка чувствительности** детектора движения Frigate.

```yaml
motion:
  threshold: 35        # Чувствительность (1-255). Чем меньше — тем чувствительнее.
                       # 25 = очень чувствительный (реагирует на тени)
                       # 35 = нормальный
                       # 50+ = грубый (пропускает мелкие движения)
  contour_area: 60     # Минимальный размер области движения в пикселях.
                       # Меньше = реагирует на мелкие объекты (насекомые)
                       # Больше = игнорирует мелкие движения
  improve_contrast: true  # Улучшить контраст перед анализом.
                          # Помогает при плохом освещении (ночью)
```

### Как подобрать threshold:

| Ситуация | Рекомендуемый threshold |
|----------|------------------------|
| Помещение (мало шума) | 20-25 |
| Улица (нормальные условия) | 30-40 |
| Улица (много шума: ветер, дождь) | 50-70 |
| Ночная съёмка | 20-30 (+ `improve_contrast: true`) |

> **Важно:** `motion` влияет только на то, когда Frigate **начинает искать объекты** в кадре. Это не то же самое что режим записи `motion: days`.

## Секция zones (зоны детекции)

Зоны позволяют ограничить область, в которой Frigate реагирует на объекты.

```yaml
zones:
  detection_zone_cam_web:      # Название зоны (придумай сам)
    coordinates: 0,0,1,0,1,1,0,1  # Координаты зоны (см. ниже)
    loitering_time: 0          # Время нахождения в зоне для alert (секунды)
                               # 0 = сразу при появлении
                               # 5 = только если объект в зоне 5+ секунд
    friendly_name: "Зона входа" # Отображаемое имя в UI
```

### Как задаются координаты:

Координаты — это пары `x,y` от 0 до 1 (относительные, не пиксели):
- `0,0` — верхний левый угол
- `1,0` — верхний правый угол
- `1,1` — нижний правый угол
- `0,1` — нижний левый угол

```
(0,0) -------- (1,0)
  |                |
  |    Камера      |
  |                |
(0,1) -------- (1,1)
```

**Пример зоны в нижней половине кадра:**
```yaml
coordinates: 0,0.5,1,0.5,1,1,0,1
```

> **Совет:** Рисуй зоны прямо в Frigate UI (Configuration → Cameras → Edit Zone) — там можно кликать мышкой, не нужно вводить координаты вручную.

## Секция ffmpeg (настройка захвата видео)

```yaml
ffmpeg:
  hwaccel_args: []     # Аргументы аппаратного ускорения
                       # [] = отключено (для USB камер обязательно!)
                       # preset-intel-qsv-h264 = Intel Quick Sync
                       # preset-nvidia-h264 = NVIDIA GPU
  inputs:
    - path: /dev/video0           # Путь к устройству (USB камера)
      # или
      # path: rtsp://user:pass@IP:554/...  # RTSP поток (IP камера)
      
      input_args: -f v4l2 -input_format mjpeg -video_size 1280x720 -framerate 30
      # -f v4l2              = использовать Video4Linux2 (драйвер USB камер)
      # -input_format mjpeg  = формат захвата (MJPEG лучше для USB)
      # -video_size 1280x720 = разрешение захвата
      # -framerate 30        = кадров в секунду с камеры
      
      roles:
        - detect    # Использовать этот поток для детекции объектов
        - record    # Использовать этот поток для записи
```

### hwaccel_args — когда что использовать:

| Значение | Когда использовать |
|----------|-------------------|
| `[]` | USB камеры, нет GPU, по умолчанию |
| `preset-intel-qsv-h264` | Intel CPU с Quick Sync (N100, i3/i5/i7) |
| `preset-nvidia-h264` | NVIDIA GPU |
| `preset-rpi-64-h264` | Raspberry Pi |

> **Для USB камер всегда используй `hwaccel_args: []`** — иначе будет ошибка "Failed to read image from surface".

## Секция detect (настройка детекции)

```yaml
detect:
  width: 1280    # Ширина кадра для детекции (должна совпадать с разрешением потока!)
  height: 720    # Высота кадра для детекции
  fps: 10        # Сколько кадров в секунду анализировать
                 # 5-10 fps достаточно для детекции людей
                 # Больше = точнее, но больше нагрузка на CPU
  enabled: true  # Включить детекцию
```

### Разрешение detect vs разрешение захвата:

```yaml
input_args: ... -video_size 1280x720 -framerate 30
# ↑ Камера захватывает 1280x720 @ 30fps

detect:
  width: 1280    # ← Должно совпадать с video_size!
  height: 720
  fps: 10        # ← Frigate анализирует только 10 из 30 кадров
```

| Параметр | Влияет на |
|----------|-----------|
| `video_size` в input_args | Разрешение live stream и записи |
| `width/height` в detect | Разрешение для анализа объектов |
| `framerate` в input_args | FPS live stream и записи |
| `fps` в detect | FPS анализа (меньше = меньше нагрузка CPU) |

## Секция objects (отслеживаемые объекты)

```yaml
objects:
  track:
    - person   # Люди
    - car      # Машины
    - dog      # Собаки
    - cat      # Кошки
    - bicycle  # Велосипеды
    - truck    # Грузовики
    - motorcycle  # Мотоциклы
```

> **Совет:** Чем меньше объектов отслеживаешь — тем меньше нагрузка на CPU. Для домашней камеры достаточно `person`.

## Полный пример конфига с объяснениями

```yaml
auth:
  enabled: true              # Включить аутентификацию для порта 8971

mqtt:
  enabled: false             # MQTT не используем (нет брокера)

detectors:
  cpu1:
    type: cpu                # Использовать CPU для детекции (без Coral TPU)

cameras:
  home_usb:                  # Название камеры (придумай сам)
    ffmpeg:
      hwaccel_args: []       # Отключить GPU (обязательно для USB камер)
      inputs:
        - path: /dev/video0  # USB камера
          input_args: -f v4l2 -input_format mjpeg -video_size 1280x720 -framerate 30
          roles:
            - detect         # Этот поток для детекции
            - record         # Этот поток для записи
    
    detect:
      width: 1280            # Совпадает с video_size
      height: 720
      fps: 10                # Анализировать 10 кадров из 30
      enabled: true
    
    objects:
      track:
        - person             # Отслеживать только людей
        - dog
        - cat
    
    record:
      enabled: true
      continuous:
        days: 0              # Не писать постоянно
      detections:
        retain:
          days: 2            # Видео с объектами хранить 2 дня
      alerts:
        retain:
          days: 7            # Видео с объектами в зоне хранить 7 дней
    
    snapshots:
      enabled: true
      retain:
        default: 5           # Снимки хранить 5 дней
    
    motion:
      threshold: 35          # Чувствительность детектора движения
      contour_area: 60       # Минимальный размер области движения
      improve_contrast: true # Улучшить контраст (помогает ночью)
    
    zones:
      detection_zone_cam_web:   # Название зоны
        coordinates: 0,0,1,0,1,1,0,1  # Весь кадр (замени на свои координаты)
        loitering_time: 0       # Реагировать сразу
        friendly_name: "Зона детекции"
```
