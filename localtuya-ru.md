![logo](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/logo-small.png)

Кастомная интеграция Home Assistant для локального управления устройствами на базе Tuya.

Интеграция обновляет статус устройств через push-уведомления, а не через опрос — поэтому обновления происходят быстро (в том числе при ручном управлении устройством).
Интеграция также поддерживает Tuya IoT Cloud API для получения информации и local_key устройств.

**Примечание: настройка Cloud API не обязательна (LocalTuya работает и без неё), но настоятельно рекомендуется для удобного получения и автоматического обновления local_key после перепривязки устройства. Обращения к Cloud API происходят только при запуске и когда требуется обновить local_key.**

Поддерживаемые типы устройств Tuya:
* Выключатели (Switches)
* Лампочки и свет (Lights)
* Шторы и жалюзи (Covers)
* Вентиляторы (Fans)
* Климатические устройства (Climates)
* Пылесосы (Vacuums)

Мониторинг энергопотребления (напряжение, ток, ватты и т.д.) поддерживается для совместимых устройств.

> **В настоящее время поддерживаются протоколы Tuya версий 3.1–3.4.**

# Установка

Самый простой способ — установить LocalTuya через [HACS](https://hacs.xyz/).

> ⚠️ LocalTuya отсутствует в стандартном каталоге HACS. Нужно добавить репозиторий вручную:
> HACS → `⋮` → Пользовательские репозитории → `https://github.com/rospogrigio/localtuya` → категория **Интеграция**

Для ручной установки: скопировать папку `localtuya` со всем содержимым в папку `custom_components` внутри директории конфигурации Home Assistant (обычно `/config`).

---

# Использование

**Примечание: для работы LocalTuya необходимы Key и ID каждого устройства. Самый простой способ — настроить Cloud API аккаунт прямо в интеграции. Если не хотите этого делать, ключи можно получить вручную через https://github.com/codetheweb/tuyapi/blob/master/docs/SETUP.md или https://pypi.org/project/tinytuya/.**

**Примечание 2: если вы планируете заблокировать устройствам доступ в интернет, нужно также заблокировать DNS-запросы (к локальному DNS-серверу, например 192.168.1.1). Если заблокировать только исходящий интернет — устройство зависнет в "зомби-состоянии" и не будет отвечать на команды с localkey. Поэтому сначала подключите устройства при активном интернете, получите local_key каждого устройства, и только потом блокируйте доступ.**

# Добавление интеграции

**Примечание: начиная с версии v4.0.0 настройка через YAML-файлы не поддерживается. Интеграция настраивается только через config flow.**

Нажмите кнопку **+ДОБАВИТЬ ИНТЕГРАЦИЮ** на странице Настройки → Интеграции и выберите LocalTuya.

Появится страница настройки Cloud API с запросом учётных данных Tuya IoT Platform:

![cloud_setup](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/9-cloud_setup.png)

Для настройки аккаунта Tuya IoT Platform обратитесь к инструкции официальной интеграции Tuya:
https://www.home-assistant.io/integrations/tuya/

`Client ID` и `Secret` находятся в `Cloud > Development > Overview`, а `User ID` — во вкладке "Link Tuya App Account" внутри проекта:

![user_id.png](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/8-user_id.png)

> **Важно: если у вас уже есть аккаунт и IoT-проект, убедитесь что он создан после 25 мая 2021 года (из-за изменений в Tuya 2.0). Иначе нужно создать новый проект.**

![project_date](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/6-project_date.png)

После нажатия Submit первоначальная настройка завершена и интеграция добавлена.

> **Примечание: вводить Cloud API credentials не обязательно — можно поставить галочку "Do not configure a Cloud API account" и интеграция всё равно добавится.**

После добавления интеграции устройства настраиваются через кнопку Configure на странице интеграций:

![integration_configure](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/10-integration_configure.png)

# Добавление и редактирование устройства

При выборе "Add or Edit a device" появится выпадающий список обнаруженных устройств (через автообнаружение при добавлении, или список уже настроенных при редактировании). Можно выбрать устройство из списка или ввести параметры вручную через опцию "...".

> **Примечание: приложение Tuya на телефоне должно быть закрыто для надёжной работы следующих шагов.**

![discovery](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/1-discovery.png)

Если устройство выбрано из списка, нужно ввести только Friendly Name и localKey. Эти значения подтянутся автоматически если настроен Cloud API аккаунт, иначе нужно ввести вручную.

**Интервал опроса (scan interval)** — необязательный параметр, нужен только если значения энергопотребления обновляются недостаточно часто. Значения менее 10 секунд могут вызвать нестабильность.

**Manual DPS To Add** — необязательный параметр, нужен если устройство не объявляет DPS корректно до инициализации сущности. Часто можно избежать, сначала подключив устройство через приложение Tuya, закрыв приложение, и только потом добавив в интеграцию. **Примечание: любые DPS добавленные этим способом будут иметь значение -1 при настройке.**

**DPIDs to send in RESET command** — необязательный параметр. Используется когда устройство не отвечает на команды после перезагрузки питания (зомби-состояние). Обычно это происходит при блокировке интернета. Типичные значения: "18,19,20". Неправильные значения могут не вывести устройство из зомби-состояния.

После нажатия Submit соединение проверяется.

![image](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/2-device.png)

Затем нужно добавить сущности — этот шаг повторяется для каждого типа. Выберите тип сущности из выпадающего меню.
После настройки всех сущностей оставьте галочку "Do not add more entities" — это завершит процедуру.

![entity_type](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/3-entity_type.png)

Для каждой сущности нужно выбрать связанный DP. Все опции с выбором DP показывают выпадающий список доступных DP устройства с их текущими значениями — это упрощает идентификацию.

**Примечание: если устройству требуется отправить инициализирующее значение для работы, это можно настроить через опцию 'Passive entity'. Можно также указать конкретное значение инициализации.**

Пример настройки сущности типа "switch":

![entity](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/4-entity.png)

После настройки сущностей процедура завершена. Устройство можно привязать к комнате в Home Assistant.

![success](https://github.com/rospogrigio/localtuya-homeassistant/blob/master/img/5-success.png)

# Миграция с LocalTuya v.3.x.x

При обновлении с v3.x.x конфигурация автоматически мигрирует. Всё будет работать как раньше, но в разделе интеграций вместо нескольких записей LocalTuya появится одна (с количеством устройств и сущностей). После миграции можно ввести учётные данные Tuya IoT для включения поддержки Cloud API.

Если LocalTuya был настроен через YAML-файлы — все его упоминания в YAML можно удалить (кроме настроек логгера).

# Мониторинг энергопотребления

Получить данные о напряжении и токе можно двумя способами:

1) Создать отдельные сенсоры с нужными именами.
   Примечание: значения напряжения и потребления обычно включают первый десятичный знак. Нужно умножить на 0.1 для получения корректных значений.
2) Обращаться к атрибутам voltage/current/current_consumption выключателя и создать template-сенсоры.
   Примечание: эти значения уже делятся на 10 для напряжения и потребления.
3) На некоторых устройствах значения энергопотребления обновляются недостаточно часто. В таком случае установите scan interval — рекомендуется 30 секунд (менее 10 секунд могут вызвать нестабильность).

```yaml
sensor:
  - platform: template
    sensors:
      tuya-sw01_voltage:
        value_template: >-
          {{ states.switch.sw01.attributes.voltage }}
        unit_of_measurement: 'V'
      tuya-sw01_current:
        value_template: >-
          {{ states.switch.sw01.attributes.current }}
        unit_of_measurement: 'mA'
      tuya-sw01_current_consumption:
        value_template: >-
          {{ states.switch.sw01.attributes.current_consumption }}
        unit_of_measurement: 'W'
```

# Отладка

При написании баг-репорта всегда прикладывайте debug-логи. Включить их можно так:

```yaml
logger:
  default: warning
  logs:
    custom_components.localtuya: debug
    custom_components.localtuya.pytuya: debug
```

Затем откройте настройки проблемного устройства и включите "Enable debugging for this device".
