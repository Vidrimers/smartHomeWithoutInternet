## Часть 1 — Сервер (мозг системы)

Это центральное устройство, на котором работает Home Assistant. Должно быть включено 24/7.

### Рекомендованный выбор: мини-ПК

**Intel NUC или аналог (например, Beelink Mini S12 Pro)**
- Процессор: Intel N100 или выше
- ОЗУ: минимум 8 ГБ (рекомендовано 16 ГБ — для голосовых моделей Whisper)
- SSD: 128–256 ГБ
- Цена: ~80–150€
- Плюсы: тихий, экономичный (10–15 Вт), легко тянет голосовую модель Whisper medium

**Бюджетный вариант: Raspberry Pi 5 (4 ГБ)**
- Цена: ~70–80€ + корпус + карта памяти
- Минус: голосовая модель Whisper ограничена уровнем `small`, реакция медленнее

### Установка Home Assistant

Используется **Home Assistant Container на Ubuntu** — официально поддерживаемый вариант, при котором Ubuntu остаётся полноценной системой.

> ℹ️ Метод Supervised (с Add-on Store) был задепрекирован в 2025.6 и официально снят с поддержки в 2025.12. Теперь поддерживаются только **Home Assistant OS** и **Home Assistant Container**.

**Почему Container, а не Home Assistant OS:**
- Ubuntu остаётся полноценной для Xray VPN, Nextcloud, Matrix и всего остального
- HA работает как обычный Docker-контейнер
- Zigbee2MQTT, Whisper, Piper, ESPHome ставятся как отдельные контейнеры (чуть сложнее, но всё работает)

**Порядок установки:**
1. Установить **Ubuntu Server 22.04 LTS** на FIREBAT (или 24.04 — оба подходят)
2. Установить официальный Docker:
   ```bash
   sudo apt install -y ca-certificates curl gnupg
   sudo install -m 0755 -d /etc/apt/keyrings
   curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
   sudo chmod a+r /etc/apt/keyrings/docker.gpg
   echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
   sudo apt update
   sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
   ```
3. Создать папку для конфигурации HA и запустить контейнер:
   ```bash
   mkdir -p ~/homeassistant
   sudo docker run -d \
     --name homeassistant \
     --privileged \
     --restart=unless-stopped \
     -e TZ=Europe/Moscow \
     -v ~/homeassistant:/config \
     --network=host \
     ghcr.io/home-assistant/home-assistant:stable
   ```
4. Открыть в браузере: `http://<IP-сервера>:8123`
   ```bash
   # Узнать IP сервера:
   ip a | grep "inet " | grep -v 127
   ```

## Часть 2 — Zigbee-координатор (хаб для устройств)

USB-донгл, который подключается к серверу и создаёт Zigbee-сеть в квартире.
Именно через него лампочки, датчики и другие устройства общаются с Home Assistant.

### Рекомендованный выбор

**Sonoff Zigbee 3.0 USB Dongle Plus (модель ZBDongle-P)**
- Чип: CC2652P (стабильный, хорошо поддерживается)
- Поддерживает до 50+ устройств
- Дальность: ~30 м в помещении (Zigbee-устройства сами удлиняют сеть)
- Цена: ~15–20€
- Где купить: AliExpress, Amazon

> ⚠️ Рекомендуется подключать через USB-удлинитель 15–30 см — чтобы отнести донгл подальше
> от корпуса ПК и избежать помех от USB 3.0.

### ZBDongle-P vs ZBDongle-E — в чём разница

Sonoff выпускает два донгла, которые легко перепутать:

| | ZBDongle-P | ZBDongle-E |
|---|---|---|
| Чип | CC2652P (Texas Instruments) | EFR32MG21 (Silicon Labs) |
| Адаптер в Zigbee2MQTT | `adapter: zstack` | `adapter: ember` |
| Макс. устройств | ~50 | ~200 |
| Поддержка | Отличная, много документации | Современный, полностью поддерживается |

> ⚠️ Модель указана на коробке: `ZBDongle-P` или `ZBDongle-E(EFR32MG21)`. Если указать неправильный адаптер — Zigbee2MQTT не запустится.

### Запуск Zigbee2MQTT через Docker

Сначала узнать путь к донглу:
```bash
ls /dev/ttyACM* /dev/ttyUSB* 2>/dev/null
```

Создать конфиг (подставить правильный `adapter`):
```bash
mkdir -p ~/zigbee2mqtt/data
cat > ~/zigbee2mqtt/data/configuration.yaml << 'EOF'
homeassistant:
  enabled: true

mqtt:
  server: mqtt://localhost:1883

serial:
  port: /dev/ttyACM0      # путь из команды выше
  adapter: ember           # ember для ZBDongle-E, zstack для ZBDongle-P

frontend:
  enabled: true
  port: 8080
EOF
```

Запустить MQTT-брокер:
```bash
sudo docker run -d \
  --name mosquitto \
  --restart=unless-stopped \
  --network=host \
  eclipse-mosquitto \
  mosquitto -c /mosquitto-no-auth.conf
```

Запустить Zigbee2MQTT:
```bash
sudo docker run -d \
  --name zigbee2mqtt \
  --restart=unless-stopped \
  --network=host \
  --privileged \
  -v ~/zigbee2mqtt/data:/app/data \
  --device /dev/ttyACM0:/dev/ttyACM0 \
  ghcr.io/koenkk/zigbee2mqtt
```

Веб-интерфейс Zigbee2MQTT: `http://<IP-сервера>:8080`

Если донгл отвалился — перезапустить контейнер:
```bash
sudo docker restart zigbee2mqtt
```

### Программное обеспечение для координатора

**Zigbee2MQTT** (рекомендуется)
- Запускается как Docker-контейнер (см. выше)
- Веб-интерфейс для управления устройствами
- Поддерживает тысячи устройств разных брендов
- Совместимость проверяется на сайте: zigbee2mqtt.io/supported-devices

**Альтернатива: ZHA (Zigbee Home Automation)**
- Встроен в Home Assistant, не требует отдельной установки
- Проще в настройке, но меньше функций

## Часть 4 — Голосовое управление (полностью локальное)

Работает без интернета. Никакой Алисы, никакого Google Assistant.

### Как это работает

```
Вы говорите → Микрофон → Whisper (распознаёт речь) → Assist (понимает команду)
→ Home Assistant выполняет → Piper (озвучивает ответ) → Динамик
```

Всё происходит локально на вашем сервере.

### Компоненты для голосового управления

В режиме Container аддоны ставятся как отдельные Docker-контейнеры, а не через Add-on Store.

1. **Whisper** — распознавание речи (speech-to-text)
   ```bash
   sudo docker run -d \
     --name whisper \
     --restart=unless-stopped \
     -p 10300:10300 \
     -v ~/whisper:/data \
     rhasspy/wyoming-whisper \
     --model small-int8 --language ru
   ```
   > Для мини-ПК (N100) можно использовать `medium-int8` вместо `small-int8` — качество лучше.

2. **Piper** — синтез речи (text-to-speech)
   ```bash
   sudo docker run -d \
     --name piper \
     --restart=unless-stopped \
     -p 10200:10200 \
     -v ~/piper:/data \
     rhasspy/wyoming-piper \
     --voice ru_RU-ruslan-medium
   ```

3. **Assist** — встроенный голосовой ассистент
   - Уже встроен в HA, не требует установки
   - После запуска Whisper и Piper: **Настройки → Голосовые помощники → Добавить**
   - Указать адреса: Whisper — `localhost:10300`, Piper — `localhost:10200`

### Выбор модели Whisper (качество vs скорость)

| Модель  | Качество русского | Задержка ответа | Требования к железу |
|---------|-------------------|-----------------|---------------------|
| tiny    | Слабое            | ~0.5 сек        | Raspberry Pi 4      |
| base    | Приемлемое        | ~1 сек          | Raspberry Pi 4      |
| small   | Хорошее           | ~2–3 сек        | Raspberry Pi 5      |
| medium  | Отличное          | ~2–4 сек        | Мини-ПК (N100+)     |

**Рекомендация:** если у вас мини-ПК (Beelink N100) — ставьте `medium`.
Если Raspberry Pi 4/5 — ставьте `small`.

### Устройство для голосового ввода

#### Рекомендованный выбор: ESP32-S3-BOX-3

- Небольшая коробочка с микрофоном, динамиком и экраном
- Прошивается через браузер в один клик (сайт: install.esphome.io) — пайка не нужна
- Крепится на стену или ставится на полку
- Всегда слушает ключевое слово (настраиваемое)
- Цена: ~35–45€ (AliExpress, Amazon)

#### Бюджетный вариант: M5Stack ATOM Echo

- Маленький кубик с микрофоном
- Нет динамика для голосового ответа (только выполняет команды)
- Цена: ~10–15€
- Прошивается также через браузер

#### Использование телефона

Приложение **Home Assistant** для Android и iOS имеет встроенную кнопку микрофона.
Нажали → сказали команду → HA выполнил. Без установки дополнительного оборудования.

## Часть 6 — ESP32 и ESPHome: свои устройства за копейки

ESP32 — это маленький микроконтроллер с Wi-Fi и Bluetooth, на котором можно собрать
практически любое умное устройство для Home Assistant. Работает полностью локально,
без облака, интегрируется в HA автоматически.

### Что такое ESPHome

**ESPHome** — это прошивка для ESP32/ESP8266, которая превращает голую плату в
умное устройство. Настраивается через простой YAML-файл, без программирования.
Устройство само появляется в Home Assistant через локальную сеть.

Установка через Docker: **`sudo docker run -d --name esphome --restart=unless-stopped -p 6052:6052 -v ~/esphome:/config ghcr.io/esphome/esphome`**

---

### Что можно собрать на ESP32

#### 🎙️ Голосовое устройство (уже упоминалось)

| Плата | Что нужно | Цена |
|---|---|---|
| **ESP32-S3-BOX-3** ⭐ | Ничего, всё встроено (микрофон, динамик, экран) | ~40€ |
| **M5Stack ATOM Echo** | Ничего, всё встроено (микрофон + маленький динамик) | ~12€ |
| ESP32-S3-DevKit + микрофон INMP441 | Нужна пайка | ~8€ |

Прошивается за 2 минуты через браузер: [voice.home-assistant.io](https://voice.home-assistant.io)

---

#### 🌡️ Датчик температуры, влажности, CO₂

Вместо покупки готовых Zigbee-датчиков можно собрать свой на ESP32 — дешевле и гибче.

**Популярные датчики под ESPHome:**

| Датчик | Что измеряет | Цена |
|---|---|---|
| **DHT22** | Температура + влажность | ~2€ |
| **BME280** ⭐ | Температура + влажность + давление | ~3€ |
| **SCD40** | CO₂ + температура + влажность | ~10€ |
| **BH1750** | Освещённость (люксы) | ~1€ |

Пример YAML для BME280 в ESPHome:
```yaml
sensor:
  - platform: bme280_i2c
    temperature:
      name: "Температура в комнате"
    humidity:
      name: "Влажность в комнате"
    pressure:
      name: "Давление"
```

---

#### 💡 Управление реле (свет, вентилятор, что угодно)

ESP32 + модуль реле = умный выключатель за ~5€. Вставляется в распределительную
коробку или корпус. Управляется из HA или по голосу.

**Популярные готовые устройства на ESP32/ESP8266 с ESPHome:**

| Устройство | Что делает | Цена |
|---|---|---|
| **Sonoff Basic R2** | Реле в корпусе, перепрошивается на ESPHome | ~6€ |
| **Sonoff Mini R2** | Компактное реле, влезает в подрозетник | ~8€ |
| **Shelly 1** | Реле, уже локальное из коробки (не нужна перепрошивка) | ~10€ |

> ⚠️ Sonoff-устройства продаются с родной прошивкой Ewelink (облако). Чтобы
> работали локально — нужно перепрошить на ESPHome или Tasmota через USB или OTA.

---

#### 🚨 Датчик движения / присутствия

Самый точный вариант — **mmWave-радар** (определяет даже неподвижного человека,
не только движение):

| Датчик | Особенность | Цена |
|---|---|---|
| **LD2410** ⭐ | mmWave, определяет присутствие, не только движение | ~4€ |
| **HC-SR501** | Классический PIR, дёшево | ~1€ |

ESP32 + LD2410 + корпус = датчик присутствия лучше любого Zigbee-датчика за ~8€ итого.

---

#### 🔘 Умная кнопка / выключатель

ESP32 + обычная кнопка = умный беспроводной выключатель. Можно назначить на любое
действие в HA: включить сцену, запустить автоматизацию, отправить уведомление.

---

### Какую плату ESP32 выбрать для DIY

| Плата | Особенности | Цена |
|---|---|---|
| **ESP32-C3 Super Mini** ⭐ | Маленькая, дешёвая, Wi-Fi + BLE, удобна для датчиков | ~2–3€ |
| **ESP32-S3 DevKit** | Мощнее, подходит для голоса и экранов | ~5–8€ |
| **Wemos D1 Mini (ESP8266)** | Старше, но дёшево и работает | ~2€ |
| **ESP32-S3-BOX-3** | Готовое голосовое устройство | ~40€ |

> Для датчиков и реле — **ESP32-C3 Super Mini** (~2–3€ на AliExpress).
> Для голоса — **ESP32-S3-BOX-3** или **M5Stack ATOM Echo**.

---

### Как прошить ESP32 через ESPHome (пошагово)

1. В Home Assistant установить аддон **ESPHome**
2. Открыть ESPHome Dashboard → **New Device**
3. Задать имя устройства и выбрать тип платы
4. ESPHome сгенерирует базовый YAML — добавить нужные датчики
5. Подключить плату по USB к компьютеру
6. Нажать **Install → Plug into this computer** — прошивка загрузится
7. После первой прошивки все обновления — по воздуху (OTA), без USB
8. Устройство автоматически появится в HA

---

### Итог: ESP32 vs готовые Zigbee-устройства

| | ESP32 + ESPHome | Готовые Zigbee |
|---|---|---|
| Цена | 💚 Очень дёшево (2–10€) | 💛 Дороже (8–20€) |
| Настройка | 💛 Нужно чуть разобраться | 💚 Проще из коробки |
| Гибкость | 💚 Полная свобода | 💛 Только то, что есть |
| Батарейка | ❌ Нужна розетка (или LiPo) | 💚 Работают от батарейки |
| Протокол | Wi-Fi (локально) | Zigbee (меш-сеть) |

**Вывод:** ESP32 идеально для стационарных мест (комнаты, коридор) где есть розетка.
Zigbee — для мест без розеток (датчики на дверях, окнах, батарейные выключатели).

10. Установить **Frigate** через Docker Compose на Ubuntu
11. Создать автоматизации (например: движение в коридоре → включить свет)

## Часть 8 — Управление ИК-устройствами: Broadlink RM4 Mini

Телевизор, кондиционер, проектор, аудиосистема — всё что управляется обычным пультом,
можно завести в Home Assistant через ИК-бластер. Работает полностью локально.

### ⭐ Рекомендуемый выбор: Broadlink RM4 Mini

**Что это:** небольшое устройство с Wi-Fi и мощным ИК-передатчиком (360°).
Подключается к розетке, обучается любому пульту и управляется из HA.

**Почему именно он:**
- Интеграция **Broadlink** встроена в Home Assistant из коробки — ничего дополнительно устанавливать не нужно
- Работает **локально**, без облака (после первоначальной настройки интернет не нужен)
- Обучается любому ИК-пульту: телевизор, кондиционер, усилитель, проектор
- Радиус действия ИК: до 6–8 метров, угол 360°
- Цена: ~15–20€

---

### Как подключить к Home Assistant

1. Подключить RM4 Mini к розетке
2. Настроить Wi-Fi через приложение Broadlink (только первый раз)
3. В HA: **Settings → Devices & Services → Add Integration → Broadlink**
4. HA автоматически найдёт устройство в сети
5. Готово — устройство появится в HA

> ⚠️ После добавления в HA интернет для работы не нужен. Приложение Broadlink больше не используется.

---

### Как обучить пультам

В Home Assistant есть встроенный инструмент обучения:

1. Открыть устройство Broadlink в HA
2. Нажать **Learn Command**
3. Навести оригинальный пульт на RM4 Mini и нажать нужную кнопку
4. HA сохранит команду
5. Повторить для каждой кнопки

Сохранённые команды можно использовать в:
- Кнопках на панели управления HA
- Автоматизациях («когда датчик движения сработал — включить телевизор»)
- Голосовых командах («Алиса... то есть — включи телевизор»)

---

### Что можно подключить

| Устройство | Пример использования в HA |
|---|---|
| **Телевизор** | Включить/выключить, сменить источник, громкость |
| **Кондиционер** | Включить, задать температуру и режим |
| **Проектор** | Включить при запуске сцены «Кино» |
| **Аудиосистема** | Включить, выбрать вход |
| **Вентилятор** | Включить по датчику температуры |

---

### Пример автоматизации с кондиционером

```yaml
automation:
  - alias: "Включить кондиционер при жаре"
    trigger:
      - platform: numeric_state
        entity_id: sensor.temperatura_v_komnate
        above: 26
    action:
      - service: remote.send_command
        target:
          entity_id: remote.broadlink_rm4_mini
        data:
          command: konditsioner_vkl_22_gradusa
```

---

### Альтернативы Broadlink

| Устройство | Плюсы | Минусы | Цена |
|---|---|---|---|
| **Broadlink RM4 Pro** | Дополнительно RF 433MHz (жалюзи, ворота) | Дороже | ~30€ |
| **ESP32 + ИК-диод** | Очень дёшево, полный контроль | Нужна пайка | ~3€ |
| **Xiaomi IR Remote** | Дёшево | Хуже поддержка в HA | ~10€ |

> Если нужно управлять ещё и радиочастотными устройствами (рольставни, ворота, розетки 433MHz) — берите **RM4 Pro** вместо Mini.

## Часть 9 — Spotify в Home Assistant

Можно настроить воспроизведение музыки через Spotify прямо из Home Assistant и управлять ею голосом.

### Как это работает

```
Голос → Whisper (распознавание) → Assist (обработка) → HA → Spotify API → колонка
```

> ⚠️ Spotify требует интернет. Управление через HA работает только при наличии соединения. Сама музыка воспроизводится через подключённые колонки.

---

### Настройка Spotify в HA

1. Перейди в **Настройки → Устройства и сервисы → Интеграции**
2. Нажми «Добавить интеграцию» и выбери **Spotify**
3. Авторизуйся через приложение Spotify
4. Требуется подписка **Spotify Premium**

---

### Пример автоматизации

```yaml
automation:
  - alias: "Включить Spotify по команде"
    trigger:
      platform: conversation
      command: "Включи спотифай"
    action:
      - service: media_player.play_media
        target:
          entity_id: media_player.spotify
        data:
          media_content_type: playlist
          media_content_id: "spotify:playlist:37i9dQZF1DX..."
```

---

### Spotify Connect — самый удобный способ

Не нужен VPN и сложные настройки. В приложении Spotify на телефоне нажми на значок устройства и выбери нужную колонку дома. На улице переключай обратно на телефон — всё работает через серверы Spotify напрямую.

| Способ | Требует | Удобство | Когда использовать |
|---|---|---|---|
| **Spotify Connect** | Premium + интернет | ⭐⭐⭐⭐⭐ | Большинство случаев |
| **Голосовая команда** | HA + интернет | ⭐⭐⭐⭐ | Голос в доме |
| **VPN + веб-интерфейс HA** | VPN + интернет | ⭐⭐ | На улице через VPN |

## Часть 10 — Tuya устройства локально (LocalTuya + HACS)

Если у тебя есть Wi-Fi устройства на платформе Tuya (HIPER IoT, Tuya Smart, Smart Life и др.) —
их можно добавить в Home Assistant локально, без облака, через интеграцию **LocalTuya**.

> ℹ️ Tuya — это платформа от TUYA INC., на которой работают сотни брендов. Проверить можно
> в приложении: если в разделе "Об устройстве" написано "Производитель: TUYA INC." — это Tuya.

---

### Шаг 1 — Установить HACS

HACS (Home Assistant Community Store) — менеджер кастомных интеграций. Нужен для установки LocalTuya.

```bash
sudo docker exec -it homeassistant bash -c "wget -O - https://get.hacs.xyz | bash -"
```

Перезапустить Home Assistant:
```bash
sudo docker restart homeassistant
```

После перезапуска в HA: **Настройки → Интеграции → Добавить → HACS**
Авторизоваться через GitHub аккаунт (нужен бесплатный аккаунт).

---

### Шаг 2 — Установить LocalTuya через HACS

В HACS LocalTuya не всегда есть в стандартном каталоге — нужно добавить репозиторий вручную:

1. В HACS нажать `⋮` (три точки) → **Пользовательские репозитории**
2. Добавить: `https://github.com/rospogrigio/localtuya` → категория **Интеграция**
3. Найти LocalTuya в списке и нажать **Скачать**

После загрузки снова перезапустить HA:
```bash
sudo docker restart homeassistant
```

---

### Шаг 3 — Получить Local Key устройств

LocalTuya требует **Local Key** для локального управления устройствами. Получить его можно через Tuya IoT Platform API.

**📄 Подробная инструкция:** [localKey.md](./localKey.md)

**Краткая версия:**

1. Создать проект на [iot.tuya.com](https://iot.tuya.com):
   - Industry: Smart Home
   - Data Center: **Central Europe** (важно выбрать правильный регион!)
   - Активировать API сервисы: IoT Core, Authorization, Smart Home Basic Service

2. Привязать аккаунт Smart Life через QR-код:
   - **Devices → Link Tuya App Account**
   - Отсканировать QR-код через приложение Smart Life
   - Выбрать **Automatic Link**

3. Получить Local Key через Python скрипт:
   ```bash
   cat > /tmp/get_local_keys.py << 'EOF'
   import sys
   sys.path.insert(0, '/home/ПОЛЬЗОВАТЕЛЬ/.local/share/pipx/venvs/tinytuya/lib/python3.12/site-packages')
   import tinytuya
   
   cloud = tinytuya.Cloud(
       apiRegion="eu",  # Твой регион
       apiKey="CLIENT_ID",
       apiSecret="CLIENT_SECRET"
   )
   
   devices = cloud.getdevices()
   for device in devices:
       print(f"\n{device['name']}: ID={device['id']}, Key={device['key']}")
   EOF
   
   python3 /tmp/get_local_keys.py
   ```

4. Узнать IP адрес устройства через роутер (по MAC-адресу из скрипта)

> ⚠️ **Важно:** Если QR-код показывает "нет доступа" — попробуй другие датацентры (Central Europe, Western Europe, China). Датацентр должен совпадать с регионом аккаунта в приложении.

---

### Шаг 4 — Добавить устройство в HA

**Настройки → Интеграции → Добавить → LocalTuya**

При первом запуске появится форма Cloud API — ввести `Client ID`, `Client Secret`, `User ID` из Tuya IoT Platform. После этого LocalTuya сам найдёт устройства в сети и подтянет их ключи.

Для каждого устройства нужно указать тип сущностей (лампочка, розетка, выключатель и т.д.) и привязать DP (data points) — номера параметров устройства. LocalTuya показывает доступные DP с текущими значениями, что упрощает настройку.

> 📄 Подробная документация по LocalTuya: [localtuya-ru.md](./localtuya-ru.md) (русский) · [оригинал](https://github.com/rospogrigio/localtuya/blob/master/README.md)

---

### Сравнение: Tuya облако vs LocalTuya vs Zigbee

| | Tuya (облако) | LocalTuya | Zigbee |
|---|---|---|---|
| Интернет | ❌ Нужен | ✅ Не нужен | ✅ Не нужен |
| Сложность настройки | 💚 Просто | 💛 Средне | 💚 Просто |
| Надёжность | 💛 Зависит от серверов | 💚 Локально | 💚 Локально |
| Замена устройств | 💛 Те же Tuya | 💛 Те же Tuya | 💚 Любые Zigbee |

**Вывод:** LocalTuya хорош для устройств которые нельзя заменить (люстра, встроенный выключатель).
Для лампочек и датчиков лучше сразу брать Zigbee — проще и надёжнее.
